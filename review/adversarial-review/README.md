# adversarial-review

Get a ruthless, INDEPENDENT second-model critique of a plan, spec, ADR, PRD, or diff via the Gemini CLI, then triage the findings (valid vs. re-litigation) and optionally fold them into the specs/issues. Use when the user wants to stress-test a plan with an outside model, "red-team" / "adversarial review" / "poke holes in" a design, or sanity-check before building.

A skill for AI coding agents (Claude Code, Cursor, etc.). Drop it in your agent's skills directory — see the repo root `install.sh`.
